# MoviePilot 本地插件: 内嵌双语字幕合成

这个仓库是一个可直接作为 `MoviePilot V2` 本地插件源使用的插件仓库。

插件目录:

```text
.
├── icons/
│   └── bilingual_subtitle.svg
├── package.v2.json
└── plugins.v2/
    └── embeddedbilingualsubtitle/
        └── __init__.py
```

## 功能

- 针对带内嵌字幕的媒体文件，自动生成“上英下中”的外置双语 `srt`
- 有英文字幕和中文字幕时，直接抽取并合成双语字幕
- 只有英文字幕、没有可用中文字幕时，可调用硅基流动或其他 OpenAI 兼容接口翻译后生成中英双语字幕
- 处理失败时，记录明确失败原因，并可通过 MoviePilot 通知发送
- 支持新媒体入库自动处理
- 支持定时扫描已有媒体库
- 支持“立即运行一次”和手工命令触发
- 支持“测试翻译一次”、命令测试和插件页面按钮测试
- 支持插件页面查看最近处理记录，并可一键清空历史
- 支持后台任务队列与页面实时进度展示，适合电视剧、批量目录和原盘目录处理

## 当前实现边界

- 当前版本输出的是外置字幕文件，不直接改写媒体容器
- 当前版本只支持可抽取为文本的字幕流，例如 `subrip`、`ass`、`ssa`、`mov_text`、`webvtt`
- `PGS`、`VobSub` 这类图片字幕暂不做 OCR，遇到这类情况会提示失败原因
- 当模型返回的“译文”仍然是英文原文时，插件会自动判定为失败并降级重试，不再直接生成假双语字幕

## 典型输出文件

默认输出文件名:

```text
电影名.zh.default.srt
```

可在插件配置里修改 `字幕后缀`，例如改成:

- `zh`
- `zh-cn`
- `chi.default`

插件会输出为:

```text
电影名.<你配置的后缀>.srt
```

## 硅基流动翻译配置

当媒体只有英文字幕、没有可用中文字幕时，插件会调用你配置的翻译接口进行翻译。

默认就是硅基流动的 OpenAI 兼容地址:

```text
https://api.siliconflow.cn/v1
```

常见可用场景:

- 硅基流动
- OpenAI 官方接口
- OneAPI / New API
- Ollama 的 OpenAI 兼容接口
- 其他兼容 `/v1/chat/completions` 的服务

需要在插件中配置:

- `硅基流动 / OpenAI兼容地址`
- `硅基流动 API Key`
- `硅基流动模型`

模型名支持自定义，直接填写平台上的完整模型名称即可。

## 测试翻译功能

插件提供 3 种测试入口，都会使用当前保存的接口地址、API Key、模型名和测试文本:

- 在配置页打开 `测试翻译一次`
- 通过命令 `/embsub_test_translate`
- 在插件详情页点击 `立即测试翻译`

测试结果会保存到插件页面中，便于确认:

- 接口地址是否可达
- API Key 是否正确
- 模型名是否填写正确
- 返回格式是否满足字幕翻译场景

如果这些配置不完整，插件会明确提示:

```text
缺少可用中文字幕，且翻译配置不完整
```

## 安装方式

1. 把当前目录作为一个本地 Git 仓库保存，或者直接放在 NAS 上。
2. 在 MoviePilot 中添加本地插件源，指向这个仓库。
3. 同步插件市场后，安装 `内嵌双语字幕合成`。
4. 在插件设置中配置扫描路径、`ffmpeg/ffprobe` 路径和硅基流动参数。

## 运行前要求

- MoviePilot V2
- 宿主机可执行 `ffmpeg` 和 `ffprobe`
- 媒体文件位于本地存储

## 推荐配置

- 开启 `监听入库事件`
- 开启 `发送通知`
- 处理存量媒体时，使用 `扫描路径 + 立即运行一次`
- 如果 NAS 上模型服务稳定，建议把翻译模型设置为偏翻译能力强的模型
